Cara Simple Melindungi File Pribadi

Dipublish: 10 May 2022

Waktu baca: 4 menit

Seorang user pastinya menyimpan banyak data di komputer, namun adakalanya karena berbagai alasan tertentu kita ingin agar data yang tersimpan diprotect sehingga tidak sembarang pihak yang bisa mengakses data tersebut. Nah dalam artikel ini akan membahas secara khusus gimana cara memprotect data yang tersimpan dengan tujuan hanya satu orang aja yang bisa mengakses

First step, siapkan data yang ingin diprotect lalu taruh dalam folder khusus, tujuannya biar lebih terorganisir. Sebagai contoh misalnya data yang ingin diprotect sebagai berikut

> folder_tes

        rahasia.txt

Second step, kita akan menginstall program age untuk ngelakuin enkripsi dan dekripsi data, silahkan meluncur ke sini dan ikuti petunjuk instalasi sesuai distro kesayangan kalian. Jika udah terinstall jangan lupa periksa dulu dengan menjalankan command berikut

age

selanjutnya kita bisa jalankan command age dalam folder yg berisi file yang ingin diprotect. Program age mendukung dua metode enkripsi/dekripsi data yaitu

  • Passphrase
  • Public/Private key-pair

tiap metode punya kelebihan dan kekurangannya masing-masing, tinggal pilih salah satu yang menurut kamu paling sesuai

Passphrase

Metode ini menggunakan password sebagai kunci pengaman data, sederhananya untuk mengakses data maka kita perlu menginput password dengan benar

Untuk melakukan enkripsi atau mengubah data untuk melindungi isi aslinya bisa menggunakan command berikut

age -p rahasia.txt > rahasia.txt.enc

"rahasia.txt.enc" bebas bisa diganti nama file dan ekstensinya, misalnya jadi "protect.enc" atau "abcdef.md"

saat menjalankan perintah diatas maka program akan menanyakan password, silahkan ketik passwordnya lalu tekan enter dan lanjutkan prosesnya sampai selesai. Jika sudah maka akan muncul file baru yang bernama "rahasia.txt.enc", file ini adalah file yang sudah terenkripsi sehingga butuh password untuk melihat isi aslinya

Lalu bagaimana cara melakukan dekripsi atau mengembalikan file yang sudah terenkripsi ke bentuk aslinya? caranya pake command berikut

age -d rahasia.txt.enc > dekripsi_rahasia.txt

"dekripsi_rahasia.txt" bebas bisa diganti nama file dan ekstensinya

saat menjalankan perintah diatas maka program akan menanyakan password, silahkan ketik passwordnya dengan benar lalu tekan enter dan lanjutkan prosesnya sampai selesai. Jika sukses maka akan muncul file baru yang bernama "dekripsi_rahasia.txt", file ini adalah file yang telah terdekripsi ke bentuk aslinya

Public/Private key-pair

Berbeda dengan metode Passphrase diatas, metode ini tidak memakai password melainkan dengan private dan public keypair dalam bentuk file sebagai kunci pengaman data, sederhananya untuk mengakses data maka kita perlu memasukkan private/public key file

sebelum melakukan enkripsi/dekripsi kita harus buat file keypair dulu dengan command berikut

age-keygen -o key.txt

"key.txt" bebas bisa diganti nama file dan ekstensinya

perintah diatas akan membuat file "key.txt" yang berisi private/public keypair yang bisa digunakan untuk enkripsi/dekripsi data

Untuk melakukan enkripsi bisa menggunakan command berikut

age -e -i key.txt rahasia.txt > rahasia.txt.enc

"rahasia.txt.enc" bebas bisa diganti nama file dan ekstensinya, misalnya jadi "protect.enc" atau "abcdef.md"

sedangkan untuk melakukan dekripsi menggunakan command

age -d -i key.txt rahasia.txt.enc > dekripsi_rahasia.txt

"dekripsi_rahasia.txt" bebas bisa diganti nama file dan ekstensinya

Handling Multiple File

diatas udah dijelasin mengenai enkripsi/dekripsi data menggunakan age namun bagaimana jika kita punya banyak data yang ingin diprotect misalnya

> folder_tes

        rahasia01.jpg
        rahasia02.txt
        rahasia03.m4a
        ...

kita bisa aja lakuin enkripsi satu persatu file diatas tapi tentunya akan makan waktu yang tidak sebentar, solusi lain adalah kita masukin file-file tadi ke dalam satu archive semisal tar,zip,rar dll. Sebagai contoh disini penulis menggunakan tar archive

tar -cvf kumpulan-rahasia.tar folder_tes/

kemudian kita bisa encrypt archive "kumpulan-rahasia.tar" menggunakan age

Best Practices

  • Jika kita sudah punya data yang sudah terenkripsi dan kita ingin menghapus data asli disarankan menggunakan teknik shredding agar data aslinya tidak bisa direcover lagi

  • Gunakan password yang sulit ditebak, dan jangan lupa backup password untuk menghindari kejadian lupa password

  • Lindungi private key jangan sampai terhapus atau berubah isinya, jika memungkinkan simpan private key terpisah dari komputer

Mohon maaf, resolusi layar anda terlalu rendah.
