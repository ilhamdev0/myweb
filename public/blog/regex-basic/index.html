Regular Expression (Regex) Dasar

Dipublish: 19 Jan 2022

Waktu baca: 4 menit

Regular Expression merupakan salah satu ilmu yang harus dipelajari terutama dalam bidang IT karena sebagai programmer atau data analyst kamu pasti sering menemukan problem yang berkaitan dengan Regular Expression.

Kenapa belajar ginian?

Regular Expresion digunakan dalam banyak kasus, contohnya pada validasi input kita bisa pake Regex untuk memeriksa apakah user memasukkan email dengan format yang benar atau tidak. Contoh lainnya pada fitur comment moderation dengan bantuan Regex kita bisa memfilter komentar yang mengandung kata-kata kasar atau spam, dalam proses pengolahan data Regex dapat digunakan untuk memfilter data yang tidak diperlukan dan masih banyak lagi contoh-contoh yang lainnya.

Apa itu Regex?

Regular Expression atau disingkat Regex menurut penulis adalah sebuah kode yang dapat digunakan untuk mencari dan mengekstrak pola tertentu dari suatu teks.

Berdasarkan wikipedia Regex memiliki 3 varian rasa yaitu:

  • BRE (Basic Regular Expressions)
  • ERE (Extended Regular Expressions)
  • Perl Regex

Pada pembahasan kali ini kita akan menggunakan command "grep" pada linux untuk melakukan pattern matching pada teks menggunakan varian Basic Regular Expressions

Case sensitive

// sample.txt


Dan aku akan lebih tidak perduli
Aku mau hidup seribu tahun lagi

grep 'aku' sample.txt

perintah diatas akan mencari kata "aku" dengan posisi bebas mulai awal kalimat, ditengah maupun diakhir. Hasilnya sebagai berikut

Dan aku akan lebih tidak perduli

Perhatikan karena bersifat case sensitive maka kata "Aku" pada baris kedua sample.txt diatas tidaklah valid

Case Insensitive

// sample.txt


Dan aku akan lebih tidak perduli
Aku mau hidup seribu tahun lagi

grep '[Aa]ku' sample.txt

bisa diisi [Aa] atau [aA]

perintah diatas akan mencari kata "aku" dan "Aku" sekaligus dengan posisi bebas mulai awal kalimat, ditengah maupun diakhir. Hasilnya sebagai berikut

Dan aku akan lebih tidak perduli
Aku mau hidup seribu tahun lagi

Multi Word

// sample.txt


September ceria
11 Januari
Musim yang bahagia

grep 'Januari\|September' sample.txt

perintah diatas akan mencari kata "Januari" dan "September" sekaligus dimanapun posisinya berada. Hasilnya sebagai berikut

September ceria
11 Januari

Multi Character

// sample.txt


nyonya hidup kembali
Di depan sekali njonja menanti

grep 'n[yj]on[yj]a' sample.txt

bisa diisi [yj] atau [jy]

perintah diatas akan mencari kata dengan ejaan yang berbeda yaitu "nyonya" dan "njonja" sekaligus dimanapun posisinya berada. Hasilnya sebagai berikut

nyonya hidup kembali
Di depan sekali njonja menanti

Optional Character

// sample.txt


taukah kamu bahwa
kau begitu sempurna

grep 'kam\?u' sample.txt

perintah diatas akan mencari kata "kamu" dan "kau" dimanapun posisinya berada. Hasilnya sebagai berikut

taukah kamu bahwa
kau begitu sempurna

hal ini bisa dilakukan karena huruf "m" disini bersifat optional sehingga kata "kamu" dan "kau" dianggap valid

Word Boundary

// sample.txt


beri aku jalan
memberi itu lebih baik
jawabannya akan ku beritahu
memberikan pertolongan

grep '\bberi' sample.txt

perintah diatas akan mencari pola kata "beri" hanya jika pola kata tsb berada di awal kata dimanapun posisinya berada. Hasilnya sebagai berikut

beri aku jalan
jawabannya akan ku beritahu

kata "memberi" dan "memberikan" tidak valid karena pola kata "beri" tidak berada di awal kata

grep 'beri\b' sample.txt

perintah diatas akan mencari pola kata "beri" hanya jika pola kata tsb berada di akhir kata dimanapun posisinya berada. Hasilnya sebagai berikut

beri aku jalan
memberi itu lebih baik

kata "beritahu" dan "memberikan" tidak valid karena pola kata "beri" tidak berada di akhir kata

grep '\bberi\b' sample.txt

perintah diatas akan mencari pola kata "beri" hanya jika pola kata tsb berdiri sendiri dimanapun posisinya berada. Hasilnya sebagai berikut

beri aku jalan

kata "memberi", "beritahu" dan "memberikan" tidak valid karena pola kata "beri" mendapat imbuhan

Match Line

// sample.txt


beritahu bagaimana caranya
beri aku kesempatan
apapun akan aku beri
hendaknya kita memberikan pertolongan

grep '^beri' sample.txt

perintah ini akan mencari kata "beri" hanya jika posisinya diawal baris. Hasilnya sebagai berikut

beritahu bagaimana caranya
beri aku kesempatan

Count Pattern

// sample.txt


Perang dunia kedua terjadi tahun 1945
kenaikan mencapai 50 kali lipat
gempa bumi dengan skala 9

grep '[[:digit:]]\{4\}' sample.txt

perintah diatas akan mencari angka dengan jumlah 4 digit dimanapun posisinya berada. Hasilnya sebagai berikut

Perang dunia kedua terjadi tahun 1945

// sample.txt


Indonesia
perjuangan
internasional

grep '[[:alpha:]]\{10\}' sample.txt

perintah diatas akan mencari kata dengan jumlah huruf sebanyak 10 huruf dimanapun posisinya berada. Hasilnya sebagai berikut

perjuangan
internasional

Info!

Perhatikan disini bahwa kata "internasional" terdiri atas 13 huruf tapi kok malah tercantum dalam hasil? sebenarnya hasil valid nya adalah seperti ini

perjuangan

internasio

namun command grep tidak melakukan pemotongan terhadap teks sehingga kata "internasional" ditampilkan apa adanya

Penutup
