OOP: Polymorphism

Polymorphism merupakan salah satu prinsip dalam Object Oriented Programming (OOP) prinsip ini biasanya diajarin buat programmer pemula yang udah paham dengan konsep Class dan Object dalam OOP

Kenapa belajar ginian?

Supaya kode kita lebih mudah dipahami dan dimaintain, nah gimana cara mencapai tujuan itu? lanjutin bacanya ya hehe..

Apa itu Polymorphism?

Definisi polymorphism bisa ditemukan di berbagai literatur, tapi yang dibahas di artikel ini adalah dari sudut pandang penulis sendiri yaitu:

"POLYMORPHISM ADALAH CLASS YG MEMILIKI METHOD DENGAN NAMA DAN TUJUAN YANG SAMA TAPI BERBEDA IMPLEMENTASINYA"

udah paham kah? kalo sudah ya stop sampai disini kalo masih belum silahkan baca perumpamaan berikut, misalkan ada 3 buah transportasi sebut saja:

  • sepeda motor
  • kapal
  • pesawat

semua transportasi ini saat sedang beroperasi tentunya bisa memperlambat kecepatannya, kalo gak bisa justru malah membahayakan bagi umat manusia.. dari sini dapat disimpulkan bahwa semua transportasi ini punya yang namanya sistem pengereman atau lebih singkat disebut rem.

Akan tetapi walaupun semua transportasi diatas punya rem namun cara kerja rem yang terpasang ternyata berbeda antara transportasi satu dengan yang lainnya, tentu saja rem sepeda motor gak akan sama dengan rem kapal apalagi rem pesawat. And that's it polymorphism.

Sekarang coba kita ubah ke dalam bentuk code

class SepedaMotor {
    rem(){
        console.log("sistem rem sepeda motor: disc brake");
    }
}

class Kapal {
    rem(){
        console.log("sistem rem kapal: reverse propulsion");
    }
}

class Pesawat {
    rem(){
        console.log("sistem rem pesawat: reverse thrust + wing spoiler + disc brake");
    }
}

Contoh penggunaannya adalah seperti ini:

class Client {
    constructor(transportasi: void){
        transportasi.rem();
    }
}

const obj = new Client(new Kapal());

Bentuk Polymorphism

Terdapat 2 bentuk dari polymorphism yaitu:

  1. polymorphism dengan duck typing
  2. polymorphism dengan Inheritance

Polymorphism dengan duck typing

Contoh diatas merupakan bentuk dari polymorphism dengan duck typing. Kenapa disebut demikian? class-class diatas mungkin memiliki nama method yang sama yaitu

rem()

tapi sejatinya class-class tersebut berdiri sendiri alias tidak punya hubungan sama sekali sehingga duck typing memiliki relasi yang longgar.

Polymorphism dengan Inheritance

Bentuk ini merupakan yang paling umum ditemui di berbagai source code, dalam bentuk ini class-class saling terhubung dengan prinsip Inheritance via Normal class, Abstract class atau Interface hal ini menjadikan relasinya bersifat ketat (strict). Method yang diinherit biasanya akan di override atau di overloading oleh subclass.

Berikut adalah contoh polymorphism via interface dengan method overriding.

interface Fitur{
    rem(): void;
}

class SepedaMotor implements Fitur {
    rem(){
        console.log("sistem rem sepeda motor: disc brake");
    }
}

class Kapal implements Fitur {
    rem(){
        console.log("sistem rem kapal: reverse propulsion");
    }
}

class Pesawat implements Fitur {
    rem(){
        console.log("sistem rem pesawat: reverse thrust + wing spoiler + disc brake");
    }
}

Untuk penggunaannya sebagai berikut

class Client {
    constructor(transportasi: Fitur){
        transportasi.rem();
    }
}

const obj = new Client(new Kapal());

Kesimpulan

Prinsip Polymorphism merupakan salah satu teknik yang sering digunakan saat ngoding terutama dengan style OOP agar kode yang dihasilkan lebih terstruktur, lebih mudah dipahami dan dimaintain. Sekian dan terima kasih
